CONSTRAINTS:
--------------
constraints are rules applied to atable columns in MySQL to ensure that only valid and acurate data is stored in the database.

Why do we use constrainrs?
--------------------------
1.Maintain data integrity
2.to prevent invalid data entry
3.avoid duplicate records
4.ensures relations between tables.
5.make the data base reliable and constraint.

Types of constraints:
----------------------
1.NotNull : it does not allow null values.
2.Unique key : it does not allow duplicate values.
3.primary key : it is the combination of unique + not null.
4.composite key : it acts as key values pair.to allot primary key to two values.
5.foreign key
6.check : it checks the range between
7.default: it takes the default values

Foreign Key Constraints:
----------------------------
a foreign key is a key used to link two tables together.
a foreign key is a field or(collection of fields) in one table that referee to the primary key in another table.
the table containing the foreign key is called child table. and the table containing the candidate key is called referenced table or parent table.


What are refrential actions:
-------------------------------
refrential actions define what happens to child table data when parent table data is update or delete.
they are used with foreign key constraints.

1.On Delete Cascade:
---------------------
when a parent table row is deleted all matching child rows are automatically deleted.
